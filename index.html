A Quick Guide to Git

Welcome to a beginner's guide to git! By beginner, I mean this guide is written by a beginner for a beginner. This page is a work in progress and will also serve as Bootstrap Framework practice. With that in mind, please join me on my journey as I "commit" to learning Git.

Reach out and connect! Feel free to contact me with any questions, comments or concerns.

What is Git?

Git is a version control system used by developers. It allows you to track and record changes you make to your files.

When you are working on a project, especially if you're working with a team, it is important to save different versions of your files as your project progresses. Git is a program you run on your local machine that allows you to document changes and save versions. This way you can revert to a previous version if something breaks or you decide you do not like a recent change. After you make changes you can stage a file and then commit which will add metadata to your file and save changes to your local machine. Then it is important to also move the updated file to a place where other developers can access it, such as GitHub. You can use Git to push your file up to a repository on GitHub. Git also allows you to pull files from GitHub back to your local machine.

Git has a website where you can learn more. Download Git and start practicing!

What is GitHub?

GitHub is an open-sourced platform used by developers to host projects, store different versions and collaborate.

While Git allows you to track changes on your local machine, GitHub is a place you can store various versions of your project where other developers can access them. GitHub is open source, meaning it is free to use and open to the public. Why might you want to put your work on a platform where other developers can see it and use it? It's good to give back to the community! Share code, contribute ideas and give and receive feedback.

Check out GitHub! Make an account and start looking at other developers code and think about sharing your own. Keep in mind, employers will look at your GitHub account. They want to see what you've done and what you're working on!

Repositories

A repository is a fancy name for a folder. However, it's not just a regular folder. A repository will not only store files for your project but will also track the changes made to those files.

You'll often hear developers call them a "repo" for short. On your local machine a repo will have a .git folder where it will store the tracked changes. You can create a new repo on GitHub by selecting the New button. You will need to name your repo and then write a short description. It is important to make your repo public so other developers, team memebers or future employers can see your work. Also make sure to check the "initialize this repository with a README" box. This will allow you to clone the repo to your local machine.

To clone your repo onto your local machine first go to and open your repo on GitHub. Click on Clone or download and copy the web URL. Next open Git and use the command git clone followed by the copied link for your new repo. Now your repo will show up on your own computer as a new folder.

Git Commands

As you start to learn and use Git here are a few commonly used commands you should know. Use these commands to practice moving in and out of different folders, listing the files in that folder and checking the status of your files.

Command Description
cd ~ change to root directory
cd folder-name change to a specific folder
cd desktop change directory to desktop
cd .. move up one directory
cd ../.. move up two directories
ls list files in the current directory
git status check the status of files in current directory
rm index.html remove file from current directory
tab auto complete a command
↑ cycle up through last commands used
explorer . open folder in file explorer

Making Files

Once you have cloned your GitHub repository onto your local machine you can start adding files. You can use Git commands to make new folders and files inside your repo.

Make a new folder

First make sure you are in your desired directory. Then use the make directory command followed by a name for the folder. When naming folders do not use spaces. Move into the new folder using the change directory command followed by the folder name.

mkdir my-folder
Make a new file

Once you are in your new folder you can add a new file to the folder using the touch command followed by a name for the file. Then if you use the list command to see the new file in your folder. You can now start making changes to your new file.

touch index.html

Saving Changes

After you have made a few changes to your file you will need to stage the changes, save the new version as a commit, and then push the new version up to your GitHub repository.

Git keeps track of changes you've made to a file. When you run git status it will list the file as untracked meaning a new version has not been tracked or saved yet. In order to save the new version you will first need to stage the file. After you have staged the file use git status again and you will see the file is now listed under "changes to be committed". Next you will need to commit the file and a new version of the file will be saved on your local machine. Next you will need to push the commit up to the master branch on your GitHub repo. This way everyone else can see the changes you have made. When you push up the file Git will make a unique id that records the push so you can track the history of the file.

Stage

After you are done making changes to a file you will want to stage the file. To do this enter the add command followed by the name of the file. You will receive no output from Git after this command, which is a good thing! That means you entered the command correctly.

git add index.html
Commit

Once a file is staged it is ready to be commited. To commit the file use the commit command followed by a string in quotes that is a brief message describing the purpose of this commit. Afterwards you can check git status again to see the file is no longer staged.

git commit -m "meaningful message"
Push

After a commit has been made you will need to push this new version of your file to the master branch on GitHub. To push the file use the push command. You will receive some output from Git and the new version of your file and a new commit will appear in your GitHub repo.

git push
